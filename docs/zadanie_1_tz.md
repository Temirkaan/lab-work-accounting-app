# Практическая работа №1
## Техническое задание. Тема 19: Система учёта выполненных лабораторных работ

### Назначение проекта
Проект предназначен для учёта выполненных лабораторных работ студентов. Система должна хранить список студентов, лабораторных работ, записи сдачи и результаты проверки.

### Описание предметной области
При учебном процессе преподавателю важно видеть, кто сдал работу, какая оценка поставлена и кому нужна доработка. Без единой системы эти данные теряются или хранятся в разных файлах. Проект объединяет их в одном приложении.

### Пользователи системы
- Преподаватель - добавляет лабораторные, проверяет работы, выставляет оценки.
- Студент - сдаёт лабораторную работу.
- Куратор группы - просматривает состояние сдач.

### Функциональные требования
- Добавление студента.
- Добавление лабораторной работы.
- Фиксация сдачи лабораторной.
- Просмотр списка сдач.
- Фильтрация по статусу и группе.
- Выставление оценки.
- Отправка работы на доработку.
- Получение списка сдач через JSON API.

### Нефункциональные требования
- Система должна работать локально в учебных условиях.
- Интерфейс должен быть понятным преподавателю.
- Хранение данных должно выполняться в SQLite.
- Проект должен иметь раздельную структуру: сервер, клиент, база данных.

### Технологический стек
| Технология | Назначение |
|---|---|
| Python | Основной язык приложения. |
| Flask | Маршруты и серверная логика. |
| SQLite | Хранение студентов, работ и сдач. |
| HTML | Разметка клиентской страницы. |
| CSS | Оформление интерфейса. |

### Макет интерфейса
Главная страница содержит блок статистики, формы добавления студентов и лабораторных, форму фиксации сдачи, фильтр и карточки сданных работ.

### Предварительное описание API
| Метод и путь | Параметры | Назначение | Ответ |
|---|---|---|---|
| `GET /` | status, group | Открывает главную страницу. | HTML. |
| `POST /students` | full_name, group_name | Добавляет студента. | Redirect. |
| `POST /labs` | title, subject, max_score, deadline | Добавляет лабораторную. | Redirect. |
| `POST /submissions` | student_id, lab_work_id, comment | Фиксирует сдачу. | Redirect. |
| `POST /submissions/<id>/check` | score, comment | Выставляет оценку. | Redirect. |
| `GET /api/submissions` | status, group | Отдаёт список сдач. | JSON. |

### Модель данных
```text
students (1) -> (many) submissions
lab_works (1) -> (many) submissions
```
